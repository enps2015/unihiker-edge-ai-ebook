# RELEASE CHECKLIST

> Navegacao: [README](README.md) • [PROJECT_INDEX](PROJECT_INDEX.md)

<a id="topo"></a>

Checklist oficial de publicação de versão.

## 1. Preparação

- [ ] Escopo da versão validado.
- [ ] Pendências críticas resolvidas.
- [ ] Plano de comunicação da release definido.

## 2. Revisão editorial

- [ ] Revisão editorial completa.
- [ ] Consistência de títulos e seções confirmada.
- [ ] Padronização de nomenclatura aplicada.

## 3. Links

- [ ] Links internos verificados.
- [ ] Links externos essenciais verificados.
- [ ] Índice mestre atualizado.

## 4. Imagens

- [ ] Imagens principais revisadas.
- [ ] Placeholders indevidos removidos ou marcados.
- [ ] Organização em pastas de assets validada.

## 5. Documentação

- [ ] Documentos institucionais atualizados.
- [ ] Home da documentação revisada.
- [ ] Mapa e trilha de aprendizagem alinhados.

## 6. CHANGELOG

- [ ] Entradas da versão adicionadas.
- [ ] Alterações classificadas com clareza.
- [ ] Data de publicação validada.

## 7. Versionamento

- [ ] Número de versão definido.
- [ ] Consistência de versão entre arquivos garantida.
- [ ] Roadmap atualizado quando necessário.

## 8. GitHub Release

- [ ] Título e notas de release preparados.
- [ ] Destaques da versão resumidos.
- [ ] Links úteis incluídos.

## 9. GitHub Pages

- [ ] Conteúdo da publicação validado.
- [ ] Navegação pós-release revisada.
- [ ] Página inicial coerente com a nova versão.

## 10. PDF

- [ ] Versões finais posicionadas corretamente.
- [ ] Arquivos obsoletos tratados.
- [ ] Link principal de download validado.

## 11. Licenças

- [ ] Licença do projeto conferida.
- [ ] Materiais de terceiros revisados.
- [ ] Conformidade documental confirmada.

## 12. Aprovação final

- [ ] Maintainer responsável aprova a publicação.
- [ ] Checklist arquivado como evidência da release.
