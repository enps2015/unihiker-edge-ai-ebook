## How to Contribute

1. Create an issue describing the proposed change.
2. Open a branch with a clear name.
3. Submit a pull request with objective description and impacted files.

## Editorial Standards

- Keep chapter structure consistent.
- Preserve technical accuracy and practical focus.
- Reference sources in `references` when adding external material.

## Review Checklist

- Content reviewed for clarity.
- Terminology consistent across chapters.
- Links and file paths validated.
