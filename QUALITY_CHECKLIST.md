# QUALITY CHECKLIST

> Navegacao: [README](README.md) • [PROJECT_INDEX](PROJECT_INDEX.md)

<a id="topo"></a>

Checklists oficiais de qualidade por área.

## 1. Documentação

- [ ] Estrutura de seções coerente.
- [ ] Navegação interna clara.
- [ ] Terminologia consistente.
- [ ] Ortografia e estilo revisados.
- [ ] Referências e links internos atualizados.

## 2. README

- [ ] Mensagem principal clara na primeira dobra.
- [ ] Links centrais funcionando.
- [ ] Seções institucionais visíveis.
- [ ] Indicadores e status coerentes com o projeto.
- [ ] Chamada para contribuição atualizada.

## 3. Livro

- [ ] Versões e arquivos organizados.
- [ ] Estrutura editorial consistente.
- [ ] Elementos de apoio presentes (sumário, referências, etc.).
- [ ] Materiais complementares corretamente vinculados.
- [ ] Qualidade visual mínima atendida.

## 4. GitHub Pages

- [ ] Estrutura de navegação validada.
- [ ] Links de entrada e saída conferidos.
- [ ] Conteúdo principal publicado.
- [ ] Seções desatualizadas removidas ou marcadas.
- [ ] Página inicial alinhada ao índice mestre.

## 5. Releases

- [ ] Escopo da release definido.
- [ ] Conteúdo revisado e consolidado.
- [ ] Changelog atualizado.
- [ ] Versão aplicada de forma consistente.
- [ ] Notas de release preparadas.

## 6. Exemplos

- [ ] Catálogo organizado por objetivo.
- [ ] Nomenclatura padronizada.
- [ ] Referência cruzada com documentação.
- [ ] Estado de manutenção definido.
- [ ] Conteúdo obsoleto sinalizado.

## 7. Tutoriais

- [ ] Progressão de aprendizagem coerente.
- [ ] Estrutura didática padronizada.
- [ ] Clareza dos objetivos por tutorial.
- [ ] Links para trilhas relacionados.
- [ ] Revisão editorial concluída.

## 8. Imagens

- [ ] Nomes padronizados.
- [ ] Qualidade visual adequada.
- [ ] Alocação em pasta correta.
- [ ] Uso consistente com BRANDING.
- [ ] Recursos obsoletos arquivados.

## 9. Links

- [ ] Links internos válidos.
- [ ] Links externos ativos.
- [ ] Âncoras de navegação funcionando.
- [ ] Links duplicados evitados.
- [ ] Redirecionamentos documentados.

## 10. Licenciamento

- [ ] Arquivo de licença presente e válido.
- [ ] Materiais com autoria de terceiros atribuídos.
- [ ] Compatibilidade de uso confirmada.
- [ ] Templates e documentos institucionais alinhados.
- [ ] Política de segurança e suporte referenciada.
