# MAINTAINER GUIDE

> Navegacao: [README](README.md) • [PROJECT_INDEX](PROJECT_INDEX.md)

<a id="topo"></a>

Guia de operação para manter o projeto organizado, previsível e sustentável.

## 1. Estrutura

- Preserve a arquitetura documental definida em docs/.
- Evite criar novas áreas sem revisão de escopo.
- Garanta que todo diretório tem README com finalidade clara.
- Mantenha o PROJECT_INDEX.md atualizado após mudanças estruturais.

## 2. Versionamento

- Use versionamento semântico para marcos públicos.
- Registre mudanças relevantes no CHANGELOG.md.
- Relacione mudanças de governança com DECISION_LOG.md.

Diretriz operacional:

1. Planejar versão no roadmap.
2. Consolidar alterações em lote coeso.
3. Revisar consistência documental.
4. Publicar release com notas objetivas.

## 3. Releases

- Defina janela de release previsível.
- Priorize estabilidade documental e clareza de navegação.
- Inclua resumo de impacto para comunidade.
- Marque itens pós-release para ciclo seguinte.

Checklist mínimo de release:

- escopo aprovado;
- documentação revisada;
- links e índice mestre atualizados;
- changelog publicado;
- anúncio em canal da comunidade.

## 4. Documentação

- Trate documentação como produto principal.
- Mantenha padrão editorial e nomenclatura consistentes.
- Evite duplicação de conteúdo.
- Use links internos para aumentar navegabilidade.

## 5. Boas práticas de manutenção

- Triar issues e PRs com regularidade.
- Fechar lacunas de contexto com templates padronizados.
- Registrar decisões importantes de forma rastreável.
- Priorizar contribuições de maior impacto coletivo.
- Evitar crescimento desordenado fora do escopo.

## 6. Governança operacional

Papéis sugeridos:

- Maintainer Lead: coordenação geral e priorização.
- Maintainer Editorial: consistência de estrutura e linguagem.
- Maintainer Community: discussões, suporte e onboarding.

Ritmo recomendado:

- revisão semanal de filas (issues/PRs);
- revisão mensal de qualidade documental;
- revisão semestral de escopo e roadmap.
